Además, permite comparar la configuración real con una instalación de referencia de manera rápida y cómoda.
•"Test IO", que permite comprobar, con facilidad y rapidez, el cableado y la configuración de los módulos de una instalación, incluida la documentación de los resultados del test.

2.2.1 	Contaje
Introducción
A partir de la versión de firmware V2.1.0 del módulo existe la posibilidad de utilizar los canales 0 y 1 en el modo de operación "Contaje". Los demás canales se utilizan como entradas estándar (modo de operación DI).
Cuando los canales 0 y 1 no se utilizan en el modo de operación "Contaje" pueden utilizarse también como entradas digitales. Los canales 0 y 1 solo pueden utilizarse como entradas de contaje conjuntamente.
Al contar, los flancos de la entrada digital se registran y evalúan en correspondencia, p. ej.:
•Para contar piezas sueltas hasta un límite máximo
•Para aplicaciones con operaciones de contaje repetitivas

Si se parametrizan los canales 0 y 1 para el contaje, existen las posibilidades siguientes:
•	En caso de rebasar por exceso un límite de contaje, el comportamiento se determina:

–Entre el valor de comparación y el límite superior de contaje.

por el programa de usuario.
El sentido de contaje es siempre adelante.

Parametrización y direccionamiento
4.1 Parámetros
Activar salida DQ
Con este parámetro se define el comportamiento del bit STS_DQ en la interfaz de 
realimentación. El bit se puede asignar en el programa de usuario con una salida de 
hardware, por ejemplo; consulte Espacio de direcciones, modo de operación Contaje  (Página 27).
Selección de flancos
Con este parámetro se determina con qué flanco se cuenta.
Puede escoger entre las siguientes opciones:
•Con flanco ascendente
•Con flanco descendente
•Con flanco ascendente y descendente
Límite superior de contaje
Con este parámetro se limita el rango de contaje. Puede introducirse un valor hasta 4294967295 (232 -1).
Valor de comparación 
Con este parámetro se determina el valor de contaje con el que se conmuta la salida digital DQ (bit STS_DQ de la interfaz de realimentación) sobre la base del evento de comparación seleccionado.
Valor de arranque
Con este parámetro se determina el valor con el que empieza el contaje y se continúa ante determinados eventos. Debe cumplirse la condición siguiente:
Límite inferior de contaje < = valor de arranque < = límite superior de contaje

4.2.3 Ejemplos de contaje
En este capítulo encontrará ejemplos relacionados con el comportamiento del contador y la forma de definirlo. Las propiedades del contador se definen en la parametrización. 
Comportamiento al rebasar un límite de contaje - parar contaje 
El apartado siguiente describe el efecto del parámetro "Comportamiento al rebasar un límite de contaje = parar contaje". Para controlar el contador se utilizan los bits del byte de control de la interfaz de control; consulte el capítulo Espacio de direcciones, modo de operación Contaje  (Página 27)
El contaje empieza con el valor de contaje actual (en el byte de control se activa SW-GATE). Secuencia de bits en el byte de control de la interfaz de control 0000 1000.
Una vez rebasado por exceso el límite superior de contaje, el contaje se cancela (parar contaje), el valor de contaje salta al límite inferior de contaje = 0. El bit STS_GATE se desactiva.
Para volver a iniciar el contaje debe desactivarse y seguidamente activarse el bit SW_GATE mediante la interfaz de control. 



Parametrización y direccionamiento
4.2 Espacio de direcciones
El byte de control de la interfaz de control determina si el contaje se reinicia con el valor actual del contador o con el valor de arranque.
Secuencia de bits en el byte de control de la interfaz de control: 
•Bits 0 a 2 = "000" Arrancar con el valor de contaje actual
•Bits 0 a 2 = "010" Arrancar con el valor de arranque
Si el bit SW_GATE del byte de control de la interfaz de control se desactiva antes de alcanzar el límite superior de contaje, el contaje se detiene.

Valores de comparación
Para cada canal se especifica un valor de comparación que permite controlar el bit de realimentación STS_DQ independientemente del programa de usuario. Cuando el valor de contaje actual cumple con la condición de comparación parametrizada, se activa el bit de realimentación STS_DQ. El bit de realimentación STS_DQ puede emplearse para controlar una salida digital de un módulo de salidas digitales. 
Los valores de comparación se pueden parametrizar y modificar durante el funcionamiento mediante el programa de usuario con el juego de parámetros 0/1. 
Control de puerta
La apertura y el cierre de la puerta software (SW-GATE) definen la ventana de tiempo en la que se capturan las señales de contaje. La puerta software se controla mediante el programa de usuario.
